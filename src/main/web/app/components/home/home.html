Diversity of the Indo-Pacific Network (DIPnet) Field Information Management System (FIMS)

A primary goal of DIPnet is to develop a web-based database and infrastructure to aid collaboration and the cross-dissemination of published data. We use the DIPnet FIMS to manage and validate data for population genetic analysis - namely individual genotypes and precise sampling locations. The following workflow is used for working with metadata and sequence files in the FIMS system:

What you need to upload you data

  1. Generate a template-This option allows the user to create an excel file with the appropriate column headers. Each individual organism/sample is entered in a row in the spreadsheet and must have a unique materialsampleID that is created by the user (your unique sample name). While only four fields are required [materialSampleID, principalInvestigator, phylum and EITHER decimalLatitude AND decimalLongitude OR locality (lat/longs are preferred)], nine additional fields are recommended, and thus automatically checked. To get the most use out of your data please fill in as much of the metadata as possible or that is relevant.
  2. Prepare your fasta file (mtDNA)
    • The name of your sequences must match the materialsampleIDs in the metadata file.
    • You can include multiple taxa in a single fasta/metadata file
  3. Prepare your fastq file (next-generation sequencing)
    • FIMS will accept single and paired end read data
    • Files must be demultiplexed and trimmed of adaptors
    • Each fastq file should represent reads from a single individual
    • Names of fastq files must match the materialsampleIDs in the metadata file up to the file extension.

The actual sequence files will not be uploaded to the FIMS system. Instead the metadata file that includes entries for each individual in the dataset and a text file of the fastq file names (one per line) will be uploaded here. This process allows for the validation of the metadata file to ensure that required fields are complete, that each materialsampleID is unique, and that the materialsampleIDs match the fastq file names.
