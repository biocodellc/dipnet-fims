Diversity of the Indo-Pacific Network (DIPnet) Field Information Management System (FIMS)

A primary goal of DIPnet is to develop a web-based database and infrastructures to aid collaboration and the cross-dissemination of published data. Although Genbank has been an important repository of sequence data, it does not enforce important aspects for population analyses - namely individual genotypes and precise sampling locations and there is no central repository nor set standards for such population genetic data and associated metadata. We use the DIPnet FIMS to manage and validate data for population genetic analysis - namely individual genotypes and precise sampling locations. The following workflow is used for working with metadata and sequence files in the FIMS system:

Users can upload two types of data

  1. Published Sanger sequence data and the associated metadata
  2. Metadata associated with next-generation sequence data
    • In the process of uploading the metadata for your next-generation sequence data we create two files 1) the SRA metadata file and 2) the BioSample datasheet making uploading your data to Genbank's Short Read Archive (SRA) database faster and easier.
* Note we are not currently accepting microsatellite datasets

What you need to do before upload you data

  • Generate a template
    • This option allows the user to create an excel file with the appropriate column headers. Each individual (organism/sample) is entered in a row in the spreadsheet and must have a unique materialsampleID that is created by the user (your unique sample name). Although there are a lot of field options, only materialSampleID, principalInvestigator, phylum and EITHER decimalLatitude AND decimalLongitude OR locality (lat/longs are preferred) are required (and marker for Sanger data). Thirteen additional fields are recommended, and thus automatically checked in the default template. To get the most use out of your data please fill in as much of the metadata as possible or that is relevant.
  1. Fasta files (Sanger sequence data)
    • The name of your fasta sequences must match the materialsampleIDs in the metadata file
    • You can include multiple taxa in a single fasta/metadata file
    • Each fasta file should only include data from a single marker type (e.g. CO1, CYB, etc)
    • If you have multiple markers for the same taxa you must upload multiple fasta files for a single metadata file
    • Fasta files names should follow this format (markerabbreviation_usertaxaabbreviation.fa)
    • Marker abbreviations can be found here
    • User taxa abbreviation are user defined and can a taxonomic description
  2. Fastq files (next-generation sequencing)
    • The actual fastq sequence files will not be uploaded and stored on the FIMS system. Instead the metadata file will be uploaded and stored here.
    • The metadata file should include one entry for each individual in the dataset.
    • FIMS will accept single and paired end read data
    • Each fastq file should contain reads from a single individual
    • Names of fastq files must match the materialsampleIDs in the metadata file up to the file extension (e.g., R1.fq.gz, .1.fq, etc)
    • For validation purposes a text file of the fastq file names (one name per line and including the file extension) will be uploaded here. If you are uploading PE data there should be two file names per sample. This process ensures that required fields are complete, that each materialsampleID is unique, and that the materialsampleIDs match the fastq file names.
    • Once uploaded the FIMS system will produce two files (SRA metadata and BioSample files) for the user to ease the upload process to NCBI’s Short Read Archive (SRA)

The user will then be directed to the SRA website for file upload.
